================================================================================
                           SCALEBILL SYSTEM ARCHITECTURE
================================================================================

ScaleBill is a production-grade, multi-tenant SaaS billing platform designed for
real-time usage tracking, tiered pricing calculations, automated invoicing, and
limit-based alerting.

--------------------------------------------------------------------------------
3. CORE ARCHITECTURAL MODULES
--------------------------------------------------------------------------------

A. Real-Time Usage Ingestion & Counters:
   - High-throughput endpoint (POST /api/usage) processes incoming events.
   - For sub-millisecond updates, usage is immediately incremented in Redis using
     atomic INCRBY counters with keys structured as: `usage:{tenant_id}:{metric}:{period}`.
   - For persistence and offline batch analysis, usage logs are written asynchronously
     to MongoDB.

B. Multi-Tenant Data Isolation:
   - Every document in MongoDB holds a `tenantId` field.
   - Database operations are filtered using `tenantId` to ensure logical data isolation.
   - Composite indexes on MongoDB tables (e.g. `(tenantId, createdAt)`) ensure O(1) query filtering
     even with millions of billing events.

C. Tiered Pricing Engine:
   - Evaluates multi-tier pricing plans in memory.
   - Supports plans:
     * Starter: ₹999/mo base. Up to 1,000 API calls included, ₹1.00/call overage.
     * Pro: ₹4,999/mo base. Up to 10,000 API calls included, ₹0.50/call overage.
     * Enterprise: ₹19,999/mo base. Up to 100,000 API calls included, ₹0.20/call overage.
   - Calculations use tier boundary mappings for O(1) interval pricing lookups.

D. Automated Invoicing (Simulated Lambda Worker):
   - At the billing period end, the invoice worker computes final billing using the pricing engine.
   - Generates a PDF invoice saved locally or simulated on S3, and sends mock emails.

E. Usage Alerts Worker:
   - Checks threshold consumption (e.g., 80% or 95% of plan limits) and raises alarms.
   - Uses AWS SNS notifications (simulated locally).
